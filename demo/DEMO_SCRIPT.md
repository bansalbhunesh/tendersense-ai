# Live demo script (about 2 minutes)

Use only the three PDFs in `demo/pdfs/` generated by `generate_demo_pdfs.py` so behaviour is deterministic.

## Preconditions

- PostgreSQL running; `AI_SERVICE_URL` points to the Python service; backend on `:8080`; UI on `:5173`.
- Optional: `ANTHROPIC_API_KEY` improves criteria extraction; without it, heuristics still match these PDFs.

## Flow

1. **Register / login** as the officer.

2. **Create a tender** (any title). Open the tender workspace.

3. **Upload tender PDF:** `01_TENDER_CRPF_DEMO.pdf`  
   - **Expect:** Criteria list populated (turnover ≥ ₹5 Cr, similar projects ≥ 3, GST mandatory, ISO optional, etc.).

4. **Register two bidders:** e.g. `ACME Infrastructure` and `Beta Constructors`.

5. **Bidder A (eligible path)**  
   - Upload `02_BIDDER_ACME_ELIGIBLE.pdf` as **ca_certificate** (or first slot).  
   - Optionally upload the same file again as **gst_certificate** if your UI expects separate files (one file already contains GST + ISO + projects).  
   - **Expect:** For turnover, extracted value **≥ 5 Cr**; GST and ISO signals present; project count **≥ 3**.

6. **Bidder B (conflict path)**  
   - Upload `03_BIDDER_BETA_CONFLICT.pdf` as **ca_certificate** (and again as a second doc type if you split uploads).  
   - **Expect:** **NEEDS_REVIEW** with **CONFLICT_DETECTED** on **annual_turnover** (5.23 Cr vs 3.10 Cr in the same pack).

7. **Run evaluation**  
   - Open **Results & graph**.  
   - Walk through **one ELIGIBLE** criterion for ACME and **one NEEDS_REVIEW** for Beta with reasoning text.

8. **Audit / review (optional)**  
   - Open **Review queue** for queued items; record an **override** and show a new **audit** row.

## Talking points

- “We never silently reject — conflicts surface for officer decision.”  
- “Every verdict ties to extracted text and confidence; the graph shows criterion → verdict linkage.”  
- “Production adds Kafka, MinIO, and stronger RBAC — same audit model.”

## Troubleshooting

- If criteria are empty: tender PDF must upload first; confirm `pdfplumber` extracted text (digital text PDFs).  
- If conflict does not appear: ensure **both** turnover lines are in the **same bidder’s** uploaded document text (PDF `03` includes both).
