# Live demo script (about 2 minutes)

Use only the four PDFs in `demo/pdfs/` generated by `generate_demo_pdfs.py` so behaviour is deterministic.

## Preconditions

- PostgreSQL running; `AI_SERVICE_URL` points to the Python service; backend on `:8080`; UI on `:5173`.
- Optional: `ANTHROPIC_API_KEY` improves criteria extraction; without it, heuristics still match these PDFs.
- For the Bharat-first beat, the deterministic engine works without `ANTHROPIC_API_KEY` even on Hindi tenders — translation runs first, then the deterministic extractor.

## Flow

1. **Register / login** as the officer.

2. **Create a tender** (any title). Open the tender workspace.

3. **Upload tender PDF:** `01_TENDER_CRPF_DEMO.pdf`  
   - **Expect:** Criteria list populated (turnover ≥ ₹5 Cr, similar projects ≥ 3, GST mandatory, ISO optional, etc.).

4. **Register two bidders:** e.g. `ACME Infrastructure` and `Beta Constructors`.

5. **Bidder A (eligible path)**  
   - Upload `02_BIDDER_ACME_ELIGIBLE.pdf` as **ca_certificate** (or first slot).  
   - Optionally upload the same file again as **gst_certificate** if your UI expects separate files (one file already contains GST + ISO + projects).  
   - **Expect:** For turnover, extracted value **≥ 5 Cr**; GST and ISO signals present; project count **≥ 3**.

6. **Bidder B (conflict path)**  
   - Upload `03_BIDDER_BETA_CONFLICT.pdf` as **ca_certificate** (and again as a second doc type if you split uploads).  
   - **Expect:** **NEEDS_REVIEW** with **CONFLICT_DETECTED** on **annual_turnover** (5.23 Cr vs 3.10 Cr in the same pack).

7. **Run evaluation**  
   - Open **Results & graph**.  
   - Walk through **one ELIGIBLE** criterion for ACME and **one NEEDS_REVIEW** for Beta with reasoning text.

8. **Audit / review (optional)**  
   - Open **Review queue** for queued items; record an **override** and show a new **audit** row.

## 30-second Bharat upgrade beat

After the main flow, run a short add-on to demonstrate the Bharat-first ingestion path:

1. **Toggle UI to हिं** (top-right language switcher).
2. **Create a fresh tender** → upload `04_TENDER_BHARAT_HINDI.pdf` (Devanagari clauses for the same eligibility rules: टर्नओवर रु. 5 करोड़, 3 समान परियोजनाएँ, जीएसटी अनिवार्य, आईएसओ 9001 वांछनीय).
3. **Expect:** OCR pulls clean Devanagari text; the translation hop renders the criteria list in EN for the officer's review while preserving the Hindi source snippet on each evidence record.
4. **Run evaluation.** Verdicts apply identically — the deterministic engine is language-agnostic once translation has normalised the clauses. No `ANTHROPIC_API_KEY` is required for this beat.

Pitch line for the judges: *"Same engine, same audit trail — now in हिन्दी. The sovereign-mode toggle (`LLM_BACKEND=disabled`) removes every foreign-cloud call; only the Bhashini hop touches a third party, and that one is govt-of-India hosted."*

## Troubleshooting

- If criteria are empty: tender PDF must upload first; confirm `pdfplumber` extracted text (digital text PDFs).  
- If conflict does not appear: ensure **both** turnover lines are in the **same bidder’s** uploaded document text (PDF `03` includes both).
- If the Hindi PDF won't generate: the script downloads `NotoSansDevanagari-Regular.ttf` to `demo/.fonts/` on first run. If your build host has no internet, fetch it manually:
  ```bash
  mkdir -p demo/.fonts
  curl -fsSL -o demo/.fonts/NotoSansDevanagari-Regular.ttf \
    "https://github.com/google/fonts/raw/main/ofl/notosansdevanagari/NotoSansDevanagari%5Bwdth%2Cwght%5D.ttf"
  ```

## Throughput benchmark

`demo/benchmark.py` hammers `/v1/evaluate` with the bundled fixture (`demo/fixtures/eval_payload.json` — 4 criteria, 2 bidders) and reports throughput + latency percentiles.

```bash
# In one terminal: start ai-service (sovereign mode for deterministic numbers)
cd ai-service && LLM_BACKEND=disabled DATA_DIR="$(pwd)/data" ALLOWED_ORIGINS="*" \
  uvicorn main:app --host 0.0.0.0 --port 8081

# In another terminal: hit it
cd demo
.venv/bin/python benchmark.py --n 20 --concurrency 10
# or for a stress run
.venv/bin/python benchmark.py --n 100 --concurrency 25 --url http://localhost:8081
```

Expected order of magnitude on a laptop, deterministic backend, warm cache disabled: **single-digit-ms median, ~50–200 req/s sustained**. With `LLM_BACKEND=anthropic`, latency is dominated by the Claude API round-trip (1–4 s per request) so throughput drops to roughly `concurrency / latency_seconds` — tune `--concurrency` accordingly.
